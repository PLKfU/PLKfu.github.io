MongoDB数据库基础（1）

2020-12-22 数据库 MongoDB 评论

MongoDB数据库

关系型数据库

  • 拥有固定的表结构(字段名、字段类型、约束条件)

  • 表与表之间可以创建外键关系

  • MySQL、Oracle、PostgreSQL、SQLserver、db2、access、sqlite-

非关系型数据库

  • 没有固定的表结构
  • 数据存储以K:V键值对形式
  • redis、MongoDB、memcache

MongoDB是一个介于关系型数据库和非关系数据库之间的产品，是非关系数据库当中功能最丰富，最像关系数据库的非关系数据库。与关系型数据库最大的区别在于没有固定的行列组织数据结构。

安装

  • 下载安装

    Custom自定义安装
取消勾选(Install MongoDB Compass)

  • 将mongod.exe所在的路径添加到环境变量中

  • 针对版本不同的补全

    手动创建data文件夹
    在data文件夹内创建db文件夹(之后的数据全部存储在该文件夹下)
    在log文件夹内创建mongod.log日志文件

  • 在MongoDB文件夹内创建一个配置文件mongod.cfg

  • 执行服务的启动命令(拷贝执行 注意路径即可)管理员身份运行
    mongod –bind_ip 0.0.0.0 –port 27017 –logpath D:\MongoDB\log\mongod.log –logappend –dbpath D:\MongoDB\data\db –serviceName “MongoDB” –serviceDisplayName “MongoDB” –instal

  • 启动、停止服务端

    net start mongodb/net stop mongodb.

  • 终端mongo启动测试

数据库的增删改查

指令 MySQL MongoDB
查 show databases; show dbs
增 create database db1; use db1# 自动创建(临时存储在内存),写入数据之后自动刷到硬盘
改 alter datatbase db1`{ [default]character set <字符集名> [default]collate<校对规则名>}`
删 drop database db1; use db1之后db.dropDatabase()

集合（表）的增删改查

指令 MySQL MongoDB
增 create table t1(id int(11)); db.createCollection(‘t1’)#刷到硬盘db.t2#在内存，有数据刷入硬盘
删 drop table t1; db.t1.drop()
改 alter table t1[修改选项]; 暂无
查 show tables; show tables

文档（记录）的增删改查

指令 MySQL MongoDB
增 insert into t2 values(…)； db.t2.insert({‘name’:’ivy’})#不指定_id则自动创建
删 alter table t2 drop name db.t2.remove({…})
改 update t2 set name = ‘ivy’ where id = 1 db.t2.update({‘_id’:1},{‘name’:’kathy})#覆盖 db.t2.update({‘name’:’ivy’},{$set:{‘name’:kathy}}) #更新
查 seect * from t2; db.t2.find()
